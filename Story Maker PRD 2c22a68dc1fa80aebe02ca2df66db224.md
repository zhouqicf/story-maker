# Story Maker PRD

## 1. 产品概述 (Overview)

### 1.1 背景

5岁左右的儿童正处于想象力爆发期，渴望听故事，同时也渴望表达。家长常常面临“讲故事灵感枯竭”的困扰。利用 AIGC 技术，可以将儿童简单的碎片化想法瞬间转化为绘声绘色的图文故事。

### 1.2 产品目标

打造一款 **零门槛、极简交互、安全有趣** 的 Mobile Web App，让不识字的儿童也能通过语音交互创作属于自己的故事，并建立自己的“数字绘本馆”。

### 1.3 目标用户

- **核心用户：** 4-6岁儿童（以语音交互、看图、听故事为主）。
- **辅助用户：** 家长（负责协助保存、二次编辑、分享）。

---

## 2. 核心用户流程 (User Flow)

> 原则： 步骤最少化，全程语音引导。
> 
1. **进入首页** -> 点击巨大的“讲故事”按钮。
2. **角色选择** -> 点击图标选择主角（如：小狮子、宇航员、公主）或上传图片。
3. **主题输入** -> 按住麦克风说话（如：“我想听一个小狮子去月球吃冰淇淋的故事”）。
4. **AI 生成中** -> 展示趣味动画（如魔法棒挥舞），流式输出故事与封面。
5. **故事消费** -> 自动开始朗读故事，展示封面。
6. **保存/操作** -> 满意则放入“故事架”，不满意则“变个新魔法”（重生成）。
7. **回顾** -> 在“故事架”点击旧故事回放或编辑。

## 3. 功能详细需求 (Functional Requirements)

### 3.1 首页 (Home)

- **UI风格：** 鲜艳明快，大圆角卡片，卡通风格。
- **核心元素：**
    - **“开始创作”按钮：** 占据屏幕显眼位置，动效引导。
    - **“我的故事架”入口：** 拟物化设计（像一个真实的书架）。
    - **背景音乐：** 欢快轻柔，支持开关。

### 3.2 创作配置页 (Creation Setup)

- **选择主角 (Character Selection)：**
    - 提供 6-8 个预设热门角色（带音效，点击“老虎”会发出吼叫声）。
    - 支持“自定义”：家长可上传一张照片或输入文字，AI提取特征。
- **语音输入主题 (Voice Input)：**
    - **长按说话/点击说话：** 交互要极其简单，并在说话时有波形反馈。
    - **语音转文字 (STT)：** 实时将语音转为文字显示在屏幕上（方便家长确认），允许手动修改。
    - **引导提示：** 如果孩子不说话，系统自动语音引导：“你想讲一个关于什么的故事呢？是去海边还是去森林？”

### 3.3 故事生成页 (Generation & Consumption)

这是产品的核心体验页面。

- **生成等待状态：**
    - 避免枯燥的 Loading 转圈。使用“正在施展魔法...”的动画。
    - **生成内容：**
        1. **故事标题：** AI 自动生成。
        2. **封面图：** 根据故事内容生成的绘本风格插图（1:1 或 4:3）。
        3. **正文内容：** 300字左右，适合3分钟阅读，语言童真。
- **语音播放 (TTS)：**
    - 故事生成完毕后，**自动播放**语音。
    - 声音要求：亲切的讲故事语气（如“大姐姐”或“卡通音”），而非机械音。
    - 播放控件：只有“暂停/播放”和“重播”，按钮要极大。
- **保存机制：**
    - 生成后暂时在缓存中。
    - 提供“放入书架”（保存）按钮和“不喜欢，换一个”（重新生成）按钮。

### 3.4 故事架 (Story Shelf)

- **展示形式：** 拟物化书架，每一本书的封面就是生成的故事图。
- **交互：** 点击书籍打开故事详情页。
- **管理：**
    - 长按书籍进入“管理模式”（需简单的家长锁，如计算 1+1=2，防止误删）。
    - 支持删除。

### 3.5 二次编辑 (Secondary Editing)

- **入口：** 在故事详情页的角落（主要给家长用）。
- **功能：**
    - **修改文字：** 修正AI逻辑错误或错别字。
    - **重绘封面：** 如果封面人物崩坏，可点击“换张图”。
    - **续写：** “然后呢？”按钮，让 AI 继续写下一段（可选高级功能）。

## 4. 技术与性能需求 (Technical & Non-functional)

### 4.1 AI 模型策略

- **文本 (LLM)：** 接入 GPT-3.5/4 或 Claude，Prompt 需深度调优。
    - *System Prompt 设定：* “你是一个专业的儿童绘本作家，请用5岁孩子能听懂的简单、生动、拟声词丰富的语言创作...”
- **生图 (Image Gen)：** DALL-E 3 或 Midjourney API (或国内 Stable Diffusion 部署)。
    - *风格锁定：* 必须锁定为“卡通插画”、“绘本风格”、“色彩鲜艳”。
- **语音 (STT/TTS)：**
    - STT (输入)：Web Speech API 或 阿里云/OpenAI Whisper。
    - TTS (输出)：Azure TTS 或 OpenAI TTS (情感表现力强)。

### 4.2 内容安全 (Safety Guardrails) **(至关重要)**

- **输入过滤：** 屏蔽暴力、恐怖、成人关键词。
- **输出过滤：** 无论用户输入什么，AI 的 Prompt 必须强制加上安全锁，确保生成的故事是阳光、正向的。
    - *兜底机制：* 如果识别到敏感意图，AI 回复：“这个故事有点太吓人了，我们换个开心的吧！”

### 4.3 性能指标

- **响应速度：** 文字流式输出（首字 < 2秒），图片异步生成（允许先出文字，图片加载中显示占位符）。
- **兼容性：** 适配主流 Mobile 浏览器 (Safari, Chrome, 微信内置浏览器)。

---

## 5. UI/UX 设计建议 (Design Guidelines)

- **字体：** 使用大字号，圆体或手写体（如“方正少儿体”），字间距要宽。
- **配色：** 糖果色系（鹅黄、天蓝、草绿、淡粉），避免纯黑纯白。
- **图标：** 尽量少用文字标签，多用直观的图标（如“保存”就是一颗爱心或一本书，“播放”就是一个大喇叭）。

---

## 6. 数据埋点 (Analytics)

- **核心指标：**
    - 日活 (DAU)。
    - 每日生成故事数量。
    - 故事保存率 (衡量生成质量)。
    - 语音播放完播率。
